## 码表

Unicode：通用的文本字符集，可以在任何系统上无缝地使用。它定义了字符的代码点(code point)，但并未定义字符序列如何存储和传输。

每个字符都有一个唯一的标识符，称为字符码点（code point）

码点和码元：码点(code point)和码元(code unit)是不同的概念。

- 码点表示 Unicode 表的键，是唯一的。既可以是基本平面的，也可以是辅助平面的。
  - 基本平面：Unicode 字符集中的前 65536 个字符，包含了大多数常用的字符。
  - 辅助平面：Unicode 字符集中的后 1048576 个字符，包含了一些罕见字符和一些特殊符号。
  - 辅助平面的字符需要使用代理对（surrogate pair）来表示，即两个 16 位的码元。

## 编码

UTF(Unicode Transformation Formats):它定义了 Unicode 代码点转换为字节序列的编码规则.

- UTF-8：可变长度的编码方式,使用 1 到 4 个字节表示一个字符。它与 ASCII 编码兼容。因它对英文文本的存储效率较高,广泛用于网页和电子邮件。
- UTF-16：使用 2 个或 4 个字节（16 位或 32 位）表示一个字符。它在处理某些亚洲语言（如中文、日文和韩文）时较为高效。
- UTF-32：使用 4 个字节表示一个字符。这使得字符的处理变得简单，但体积大、存储和传输效率低。

UCS-2

- UTF-16 的一种实现，使用 2 个字节表示一个字符。它是 Unicode 标准的一部分，用于表示 Unicode 字符集中的基本平面（Basic Multilingual Plane，BMP）。
- 基本平面：Unicode 字符集中的前 65536 个字符，包含了大多数常用的字符。
- javascript 默认的处理字符编码的方式
