# sameValue 算法

ECMA 规范中定义了 “SameValue 算法”：
- 对于原始类型值：只有当类型相同且值完全一致时才返回 true。
- 对于对象：只有当两者的值指向同一个内存地址（即引用同一个对象）时才返回 true。
- 特殊值处理：对 NaN、-0 和 +0 的判断更符合数学逻辑（见下文对比）。

Object.is方法使用了SameValue算法

## `Object.is` 与 `===`的区别
核心区别在于特殊值的处理

|比较场景  |`Object.is(a, b)`  |`a === b`  |
|---------|---------|---------|
|NaN 与 NaN     | true        | false      |
|-0 与 +0     |  false       |  true       |

Object.is 更加符合预期

两者在其余方面没有差别：
- 先比较类型
- 再比较值
